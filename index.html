1 The challenges

Problems the tools address

Four practical problems shaped every tool below.

Consistency

Every unit is built differently

Curriculum comes from many hands, so teacher decks, guides, and student documents vary in look and structure — quality depends on who authored each unit.

Addressed by a shared unit template · what's next →
Source of truth

Documents anyone can edit

Files live on SharePoint open to edits by anyone with access, so there's no protected, authoritative version of a unit to rely on.

Addressed by a single published version · what's next →
Clarity

A lot to hold at once

A long teacher guide, a slide deck, and student handouts are hard to synthesize while actually running a room.

Addressed by interactive guides & the concept map · teacher tools →
Insight

Little formative signal

We currently get very little data on how students are reasoning through the work as it happens — feedback arrives late, if at all.

3 What's next

Where it's heading

The direction the toolset is going — building on what's already working.

Consistency

One shared unit template

Every unit's materials look and work the same way no matter who builds them — so consistency and quality stop depending on which teacher authored it.

Trust

A single source of truth

A published, canonical version of each unit that can't be quietly edited out from under the team.

Insight

Quiet, exam-free insight into student thinking

Building on what the Skills & Careers tool already proves possible — capturing how students reason as a byproduct of the work, never a test.

For Students

Self-paced tools students use directly — each one lets a mixed-experience class start where they are.

Stadium Logistics Rover — Companion

Shipped

A self-paced guide for the rover coding challenge — students pick their own entry point, from drag-and-drop blocks all the way up to real Python.

Lets a mixed-experience class each start where they are and pull just-in-time hints, instead of everyone stalling at the same wall waiting on the teacher.

Open tool →

River Crossing — Companion

Shipped

The same self-paced robot-coding companion, built around the classic wolf–goat–cabbage logic puzzle.

Pairs a reasoning puzzle with the code so students debug their thinking before their program; tiered hints keep them moving without hand-holding.

Open tool →

Engineering Skills & Careers

Shipped

A reflection tool where students star the lessons and careers that hooked them, rank a top three, and name a dream job.

Connects hands-on work to real career paths and captures interest as anonymous totals — no names — so the program can see what lands without surveilling anyone.

Open tool →

For Teachers

Tools that support planning, delivery, and coaching — one reference to run the room from.

Hydraulic Machines / JudoBot — Teacher Guide

Shipped

A full interactive guide for the 2-day JudoBot hydraulics build: phase-by-phase timing, prep checklist, materials, and talking points.

Replaces a long static document with a paced, in-class reference a teacher can run the room from.

Open tool →

Hydraulic Rescue Arm — Teacher Guide

Shipped

A teacher guide for the 3-day Hydraulic Rescue Arm challenge — force ratios, build steps, and the relay competition.

One structured, navigable guide in place of a sprawling multi-page document.

Open tool →

Curriculum Concept Map

In Progress

An interactive map of how every lesson connects to every other — by shared concepts, skills, and build progressions — rather than just by calendar order.

Makes the hidden structure of the year visible, so planning and coaching can build on the threads between lessons instead of treating each one in isolation.

For Leadership

Built for program oversight and decisions — aggregate signal, never personal data.

Engineering Skills & Careers — Results Dashboard

Shipped

A live, anonymous rollup of which lessons and careers students gravitate toward across the program.

Gives leadership a real signal on what's resonating across the program — aggregate only, no personal data.

Open tool →

Year-in-Numbers Data Dashboard

In Progress

Five data lenses on a full grade-year of lesson design — concept frequency, coverage gaps, and alignment to national engineering-practice standards.

Turns a year of curriculum into evidence leadership can actually see, so coverage and balance decisions rest on data instead of intuition.
